Nivel 2 - Medio

Exposicion de informacion, rutas sin autenticacion y datos internos visibles desde el navegador.

Ver recursos del nivel

Reglas de ciber del nivel

  • Prueba primero sin sesion iniciada.
  • Distingue autenticacion de autorizacion: estar logueado no significa ser admin.
  • Marca como sensible cualquier secreto, ruta local, token, host interno o password.
  • Una pagina que solo "muestra datos" tambien puede ser una vulnerabilidad.

Broken access control

Entrar a admin sin permiso

Abrir admin

Objetivo: comprobar si una ruta sensible esta protegida por login y rol.

Pasos

  1. Cierra sesion si estabas logueado.
  2. Abre /admin directamente.
  3. Revisa si ves usuarios, passwords o secretos simulados.

Que observar

Si carga sin login, el control de acceso esta roto.

Riesgo

Un atacante puede recolectar credenciales o datos internos sin explotar nada mas.

Defensa: exigir sesion y validar rol admin en servidor antes de consultar datos sensibles.

Information disclosure

Leer informacion interna

Abrir debug

Objetivo: identificar informacion tecnica que facilita ataques posteriores.

Pasos

  1. Abre /debug-info.
  2. Busca secret key, rutas locales y variables de entorno.
  3. Decide que informacion deberia ocultarse.

Checklist sensible

  • Secret keys
  • Paths internos
  • Hosts privados
  • Usuarios actuales
Defensa: apagar debug publico, usar errores genericos y guardar secretos en variables de entorno no expuestas.
