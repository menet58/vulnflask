  • No confundas validacion visual con seguridad del servidor.
  • La defensa real debe estar en backend: parametros, hashes, rate limit y CSRF.

SQL Injection

Bypass de autenticacion

Abrir login

Objetivo: saltar el login manipulando la consulta SQL que arma el servidor.

Pasos

  1. Abre /login.
  2. Usa el payload en el campo usuario.
  3. Escribe cualquier password y envia el formulario.
  4. Confirma si llegas al dashboard.

Payload

admin' --

Que observar

El comentario SQL corta la parte del password y puede devolver la fila del usuario admin.

Defensa: usar consultas parametrizadas, passwords hasheados y errores genericos.

Debug leak

Leer la consulta generada

Abrir login

Objetivo: ver como una fuga de debug facilita construir un payload correcto.

Pasos

  1. Intenta login con datos falsos.
  2. Observa el bloque DEBUG QUERY.
  3. Agrega comillas y comentarios SQL para ver como cambia la consulta.

Payload alternativo

' OR '1'='1' --

Que observar

La app muestra la estructura exacta del SQL vulnerable.

Defensa: no mostrar queries internas al usuario y enviar logs sensibles solo a un sistema seguro.
